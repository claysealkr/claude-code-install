---PAGE_TITLE---
Install Claude Code on Mac

---PAGE_SUBTITLE---
A step-by-step guide — no experience needed

---STEP---
TITLE: Open Terminal
STEPS:
- Press Command + Space to open Spotlight Search
- Type Terminal and press Return
- A window with a text prompt will appear — this is normal
TIP: Terminal is a built-in Mac app. You're not breaking anything by opening it.

---STEP---
TITLE: Check What You Already Have
INTRO: You may already have the software you need. Run this command in Terminal to check:
COMMAND: node --version
BRANCH_YES: Got a version number? (e.g. v20.0.0) — You already have Node.js. Skip ahead to Install Claude Code.
BRANCH_NO: Got an error? Run this to check if you have Homebrew: brew --version
  - Got a version number? Run "brew install node" then skip to Install Claude Code.
  - Got another error? Continue to the next step to install Node.js.

---STEP---
TITLE: Install Node.js (skip if you already have it)
INTRO: Node.js is a runtime that lets you run JavaScript outside of a browser. Claude Code is built on it, so it needs to be installed on your machine.
STEPS:
- Go to https://nodejs.org
- Click the big green "Download Node.js (LTS)" button
- Open the downloaded .pkg file
- Follow the installer — click Continue, Agree, and Install
- Enter your Mac password if prompted, then click Close when done
TIP: LTS stands for "Long Term Support" — it's the most stable version.

---STEP---
TITLE: Install Claude Code
INTRO: Copy and paste this command into Terminal, then press Return:
COMMAND: npm install -g @anthropic-ai/claude-code
TIP: A lot of text will scroll by — that's normal. Wait until the prompt reappears before continuing.

---STEP---
TITLE: Get an API Key
INTRO: An API key is a unique password that lets Claude Code authenticate as you when making requests to the AI model. You'll generate one through the KrAIG UI — Kroger's internal portal for managing AI access.
STEPS:
- Navigate to the KrAIG UI
- Select the “+ Create New Key” button

- Fill in the following Fields:
   - Owned by: “You”
   - Team: You can skip this field, it is not required for Claude Code usage
   - Key Name: “claude-code-{firstname}-{lastname}”, for example: claude-code-josh-boldt
   - Models: “All Team Models”
   - Click the “Optional Settings” bar and fill in the following fields:
      - Tags: “claude-code-wave-2” (type in then hit Enter on your keyboard if not selectable from the dropdown)
      - Under “Key Lifecycle”: Expire Key: “365d” (365 days)
   - Click “Create Key” at the bottom
- Your key should appear in a popup
- Select the “Copy API Key” button and save your API Key to reference later. It will not be shown to you again.
- If needed, you can regenerate your API key in case it gets compromised or you lose it. Search for your key alias. After clicking your key, the next page has a button to regenerate.
TIP: The key starts with sk-... — save it somewhere safe. You won't be able to see it again.

---STEP---
TITLE: Configure Claude Code
STEPS:
- Open Terminal if it is not already open
- Open (or create) the file ~/.claude/settings.json in TextEdit. Make sure the file is a Plain Text Document, not a Rich Text Document (.rtf).
- Paste in the configuration below, replacing <replace_with_your_apikey> (including the angle brackets) with the API key you created in the previous step
- Save the file
CONFIG:
{
  "apiKeyHelper": "echo <replace_with_your_apikey>",
  "autoUpdatesChannel": "stable",
  "env": {
    "ANTHROPIC_BASE_URL": "https://ai-model-proxy.aks-ur-prd-internal.8451.cloud",
    "DISABLE_AUTOUPDATER": 1,
    "DISABLE_ERROR_REPORTING": 1,
    "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": 1
  },
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "WebSearch"
    ]
  },
  "includeCoAuthoredBy": true,
  "companyAnnouncements": [
    "Welcome to Claude Code for 84.51º and Kroger! Please remember not to share your API key with others.",
    "If you encounter any issues, please email the Claude Code support DL: ClaudeCodeSupport_DL@8451.com"
  ]
}
TIP: If the ~/.claude/ folder does not exist yet, create it. The ~ symbol means your home folder (this should be your EUID).
TIP: The .claude folder is hidden by default in Finder. To reveal hidden folders, open Finder and press Command (⌘) + Shift + Period (.).

---STEP---
TITLE: Set Up SSL Certificates
INTRO: Zscaler intercepts network traffic and requires its own certificates to be trusted. Without this step, Claude Code may fail to connect to MCP servers.
STEPS:
- Copy the script below and paste it into Terminal, then press Return
- To make this permanent, add this line to your ~/.zshrc file: export NODE_EXTRA_CA_CERTS=~/.certs/all.pem
- Close and reopen Terminal to pick up the new settings
SCRIPT:
#!/bin/bash

function setup-certs() {
  local cert_path
  cert_path="$HOME/.certs/all.pem"
  local cert_dir
  cert_dir=$(dirname "${cert_path}")
  [[ -d "${cert_dir}" ]] || mkdir -p "${cert_dir}"

  security find-certificate -a -p /System/Library/Keychains/SystemRootCertificates.keychain >"${cert_path}"
  security find-certificate -a -p /Library/Keychains/System.keychain >>"${cert_path}"

  export GIT_SSL_CAINFO="${cert_path}"
  export AWS_CA_BUNDLE="${cert_path}"
  export NODE_EXTRA_CA_CERTS="${cert_path}"

  if [ ! -f "$HOME/.kroger-certs-set" ]; then
    npm config set -g cafile "${cert_path}"
    npm config set -g strict-ssl true
    yarn config set cafile "${cert_path}" -g
    yarn config set strict-ssl true -g
  fi

  touch $HOME/.kroger-certs-set
}

export NODE_OPTIONS=--use-openssl-ca

setup-certs
TIP: You only need to run this script once. The ~/.zshrc line ensures the certificate path is remembered every time you open a new Terminal window.

---STEP---
TITLE: Start Claude Code
INTRO: In Terminal, type this and press Return:
COMMAND: claude
BODY: The first time you run it, it will ask for your API key. Paste the key from the previous step and press Return.

---STEP---
TITLE: Connect the Figma MCP Server (Optional)
INTRO: Claude Code can connect to Figma to read and implement designs directly. This requires a Figma Personal Access Token and a one-time config file.
STEPS:
- Go to figma.com → click your profile icon (top right) → Settings → Security → scroll to Personal access tokens → Generate new token
- Give it a name (e.g. claude-code) and copy the token — it will not be shown again
- Open (or create) the file ~/.claude/mcp.json in a text editor
- Paste in the configuration below, replacing <replace_with_your_figma_token> (including the angle brackets) with your token
- Save the file and restart Claude Code
CONFIG:
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "figma-developer-mcp", "--stdio"],
      "env": {
        "FIGMA_API_KEY": "<replace_with_your_figma_token>"
      }
    }
  }
}
TIP: To verify it's working, start Claude Code — you should see "figma" listed under connected MCP servers at startup.

---STEP---
TITLE: Create a GitHub Account (Optional)
INTRO: GitHub is where developers store and share code. Claude Code works well with GitHub for managing projects.
STEPS:
- Go to https://github.com/signup
- Enter your Kroger email address, create a password, and choose a username
- Complete the verification steps and confirm your email address
- Once logged in, run the following commands in Terminal to connect git on your Mac to your account (replace the values in quotes with your own):
COMMAND: git config --global user.name "Your Name"
COMMAND2: git config --global user.email "you@example.com"
- Download GitHub Desktop for a visual interface to manage your code:
  - Go to https://desktop.github.com and click "Download for macOS"
  - Open the downloaded file and drag GitHub Desktop to your Applications folder
  - Open GitHub Desktop and sign in with your GitHub account
TIP: Use the same email address you signed up with so your commits are linked to your account.

---FINAL---
TITLE: Using Claude Code Going Forward
STEPS:
- Open Terminal (Command + Space, type Terminal)
- Type cd (with a space), then drag your project folder into the Terminal window and press Return
- Type claude and press Return
TIP: You only need to install everything once. Next time, just open Terminal and type claude.
