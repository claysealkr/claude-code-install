Install Claude Code on Mac

A step-by-step guide — no experience needed

Open Terminal

  1. Press Command + Space to open Spotlight Search
  2. Type Terminal and press Return
  3. A window with a text prompt will appear — this is normal
Note: Most of the time, we use apps to interact with our computer — they're a friendly layer that makes things easier. Terminal skips that layer and lets you talk to your computer directly. It's more powerful, but also more literal: you have to tell it exactly what to do.

Install Homebrew and Node.js

Homebrew is a package manager for Mac — it lets you install developer tools from the command line. Node.js is also required for some optional features like the Figma integration.

  1. Copy and paste this command into Terminal to install Homebrew, then press Return:
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Follow the prompts — it may ask for your Mac password. When it finishes, you may be shown a "Next steps" section with commands to add Homebrew to your PATH. Run those commands if shown.
  3. Confirm Homebrew installed correctly:
    brew --version

    You should see something like: Homebrew 4.x.x

  4. Then install Node.js:
    brew install node
  5. Confirm Node.js installed correctly:
    node --version

    You should see something like: v22.x.x

Tip: Both commands should print a version number. If you already have Homebrew or Node.js, the install commands will just confirm it.

Install Claude Code

Copy and paste this command into Terminal, then press Return:

curl -fsSL https://claude.ai/install.sh | bash
Note: A lot of text will scroll by — that's normal. Also... it may look like it's doing nothing. Be patient. Wait until a new line appears where you can type again — it will look something like username@machine ~ % — before continuing.

Get an API Key

An API key is a unique password that lets Claude Code authenticate as you when making requests to the AI model. You'll generate one through the KrAIG UI — Kroger's internal portal for managing AI access.

  1. Navigate to the KrAIG UI
  2. Select the "+ Create New Key" button
Important: The key starts with sk-... — save it somewhere safe. You won't be able to see it again.
  1. Fill in the following fields:
    • Owned by: "You"
    • Team: You can skip this field, it is not required for Claude Code usage
    • Key Name: claude-code-{firstname}-{lastname}, for example: claude-code-josh-boldt
    • Models: "All Team Models"
    • Click the "Optional Settings" bar and fill in:
      • Tags: claude-code-wave-2 (type in then hit Enter on your keyboard if not selectable from the dropdown)
      • Under "Key Lifecycle" — Expire Key: 365d (365 days)
    • Click "Create Key" at the bottom
  2. Your key should appear in a popup
  3. Select the "Copy API Key" button and save your API Key to reference later. It will not be shown to you again.
  4. If needed, you can regenerate your API key in case it gets compromised or you lose it. Search for your key alias. After clicking your key, the next page has a button to regenerate.

Configure Claude Code

  1. Open Terminal if it is not already open
  2. Open (or create) the file ~/.claude/settings.json in TextEdit. Make sure the file is a Plain Text Document, not a Rich Text Document (.rtf).
Note: The .claude folder is hidden by default in Finder. To reveal hidden folders, open Finder and press ⌘ + Shift + .
  1. Paste in the configuration below, replacing <replace_with_your_apikey> (including the angle brackets) with the API key you created in the previous step:
{ "apiKeyHelper": "echo <replace_with_your_apikey>", "autoUpdatesChannel": "stable", "env": { "ANTHROPIC_BASE_URL": "https://ai-model-proxy.aks-ur-prd-internal.8451.cloud", "DISABLE_AUTOUPDATER": 1, "DISABLE_ERROR_REPORTING": 1, "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": 1 }, "permissions": { "deny": [ "Read(./.env)", "Read(./.env.*)", "WebSearch" ] }, "includeCoAuthoredBy": true, "companyAnnouncements": [ "Welcome to Claude Code for 84.51º and Kroger! Please remember not to share your API key with others.", "If you encounter any issues, please email the Claude Code support DL: ClaudeCodeSupport_DL@8451.com" ] }
  1. Save the file

Set Up SSL Certificates

Zscaler intercepts network traffic and requires its own certificates to be trusted. Without this step, Claude Code may fail to connect to MCP servers.

  1. Run this command first to check if the directory exists:
    ls $HOME/.certs/
  2. If the above shows an error, the directory doesn't exist — run the following commands one by one:
    sudo mkdir -p $HOME/.certs/
    sudo security find-certificate -a -p /System/Library/Keychains/SystemRootCertificates.keychain > $HOME/.certs/all.pem
    sudo security find-certificate -a -p /Library/Keychains/System.keychain >> $HOME/.certs/all.pem
    export GIT_SSL_CAINFO=$HOME/.certs/all.pem
    export AWS_CA_BUNDLE=$HOME/.certs/all.pem
    export NODE_EXTRA_CA_CERTS=$HOME/.certs/all.pem
  3. To make this permanent, add the following line to your ~/.zshrc file:
    • If the file doesn't exist, run this to create it and open it in TextEdit:
      touch ~/.zshrc && open -e ~/.zshrc
    • Add this line to the file, then save:
      export NODE_EXTRA_CA_CERTS=~/.certs/all.pem
  4. Close and reopen Terminal to pick up the new settings
Note: You only need to run this script once. The ~/.zshrc line ensures the certificate path is remembered every time you open a new Terminal window.

Start Claude Code

In Terminal, type this and press Return:

claude

Connect the Figma MCP Server (optional)

Claude Code can connect to Figma to read and implement designs directly. The easiest way is via the official Figma plugin.

  1. In Terminal, run:
    claude plugin install figma@claude-plugins-official
  2. Start a new Claude Code session:
    claude
  3. Type /mcp in Claude Code, select figma, then choose Authenticate
  4. A browser window will open — click Allow Access
  5. Type /mcp again to confirm — you should see: Authentication successful. Connected to figma
If the plugin install doesn't work: Try the manual method instead. In Terminal, run:
claude mcp add --transport http figma https://mcp.figma.com/mcp
Then follow steps 2–5 above.

Create a GitHub Account (optional)

GitHub is where developers store and share code. Claude Code works well with GitHub for managing projects.

  1. Go to github.com/signup
  2. Enter your Kroger email address, create a password, and choose a username
  3. Complete the verification steps and confirm your email address
  4. Once logged in, run the following commands in Terminal to connect git on your Mac to your account (replace the values with your own):
    git config --global user.name "Your Name"
    git config --global user.email "you@example.com"
  5. Download GitHub Desktop for a visual interface to manage your code:
    • Go to desktop.github.com and click "Download for macOS"
    • Open the downloaded file and drag GitHub Desktop to your Applications folder
    • Open GitHub Desktop and sign in with your GitHub account
Tip: Use the same email address you signed up with so your commits are linked to your account.

Using Claude Code Going Forward

  1. Open Terminal (Command + Space, type Terminal)
  2. Type claude and press Return
  3. If you're continuing work on an existing project, just tell Claude what you'd like to work on — for example, "I want to work on the deliveryops project"
  4. If you're starting something new or need Claude to read files directly, navigate to your project folder first: type cd (with a space), drag the folder into the Terminal window, and press Return — then type claude
Tip: You only need to install everything once. Next time, just open Terminal and type claude.
