Install Claude Code on Mac

A step-by-step guide — no experience needed

Open Terminal

  1. Press Command + Space to open Spotlight Search
  2. Type Terminal and press Return
  3. A window with a text prompt will appear — this is normal
Tip: Terminal is a built-in Mac app. You're not breaking anything by opening it.

Check What You Already Have

You may already have the software you need. Run this command in Terminal to check:

node --version

Then find your result below:

Got a version number? (e.g. v20.0.0)

You already have Node.js. Skip ahead to Install Claude Code.

Got an error? Run this to check if you have Homebrew:

brew --version
  • Got a version number? Run brew install node then skip to Install Claude Code.
  • Got another error? Continue to the next step to install Node.js.

Install Node.js (skip if you already have it)

  1. Go to https://nodejs.org
  2. Click the big green "Download Node.js (LTS)" button
  3. Open the downloaded .pkg file
  4. Follow the installer — click Continue, Agree, and Install
  5. Enter your Mac password if prompted, then click Close when done
Note: LTS stands for "Long Term Support" — it's the most stable version.

Install Claude Code

Copy and paste this command into Terminal, then press Return:

npm install -g @anthropic-ai/claude-code
Note: A lot of text will scroll by — that's normal. Wait until the prompt reappears before continuing.

Get an API Key

  1. Navigate to the KrAIG UI
  2. Select the "+ Create New Key" button
  3. Fill in the following fields:
    • Owned by: "You"
    • Team: You can skip this field, it is not required for Claude Code usage
    • Key Name: claude-code-{firstname}-{lastname}, for example: claude-code-josh-boldt
    • Models: "All Team Models"
    • Click the "Optional Settings" bar and fill in:
      • Tags: claude-code-wave-2 (type in then hit Enter on your keyboard if not selectable from the dropdown)
      • Under "Key Lifecycle" — Expire Key: 365d (365 days)
    • Click "Create Key" at the bottom
  4. Your key should appear in a popup
  5. Select the "Copy API Key" button and save your API Key to reference later. It will not be shown to you again.
  6. If needed, you can regenerate your API key in case it gets compromised or you lose it. Search for your key alias. After clicking your key, the next page has a button to regenerate.
Important: The key starts with sk-ant-... — save it somewhere safe. You won't be able to see it again.

Configure Claude Code

  1. Open Terminal if it is not already open
  2. Open (or create) the file ~/.claude/settings.json in a text editor
  3. Paste in the configuration below, replacing <replace_with_your_apikey> (including the angle brackets) with the API key you created in the previous step:
{ "apiKeyHelper": "echo <replace_with_your_apikey>", "autoUpdatesChannel": "stable", "env": { "ANTHROPIC_BASE_URL": "https://ai-model-proxy.aks-ur-prd-internal.8451.cloud", "DISABLE_AUTOUPDATER": 1, "DISABLE_ERROR_REPORTING": 1, "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": 1 }, "permissions": { "deny": [ "Read(./.env)", "Read(./.env.*)", "WebSearch" ] }, "includeCoAuthoredBy": true, "companyAnnouncements": [ "Welcome to Claude Code for 84.51º and Kroger! Please remember not to share your API key with others.", "If you encounter any issues, please email the Claude Code support DL: ClaudeCodeSupport_DL@8451.com" ] }
  1. Save the file
Tip: If the ~/.claude/ folder does not exist yet, create it. The ~ symbol means your home folder (the one with your name on it).
Note: The .claude folder is hidden by default in Finder. To reveal hidden folders, open Finder and press ⌘ + Shift + .

Set Up SSL Certificates

Zscaler intercepts network traffic and requires its own certificates to be trusted. Without this step, Claude Code may fail to connect to MCP servers.

  1. Copy the script below:
#!/bin/bash function setup-certs() { local cert_path cert_path="$HOME/.certs/all.pem" local cert_dir cert_dir=$(dirname "${cert_path}") [[ -d "${cert_dir}" ]] || mkdir -p "${cert_dir}" security find-certificate -a -p /System/Library/Keychains/SystemRootCertificates.keychain >"${cert_path}" security find-certificate -a -p /Library/Keychains/System.keychain >>"${cert_path}" export GIT_SSL_CAINFO="${cert_path}" export AWS_CA_BUNDLE="${cert_path}" export NODE_EXTRA_CA_CERTS="${cert_path}" if [ ! -f "$HOME/.kroger-certs-set" ]; then npm config set -g cafile "${cert_path}" npm config set -g strict-ssl true yarn config set cafile "${cert_path}" -g yarn config set strict-ssl true -g fi touch $HOME/.kroger-certs-set } export NODE_OPTIONS=--use-openssl-ca setup-certs
  1. Open Terminal and paste the script, then press Return
  2. To make this permanent, add the following line to your ~/.zshrc file:
    export NODE_EXTRA_CA_CERTS=~/.certs/all.pem
  3. Close and reopen Terminal to pick up the new settings
Note: You only need to run this script once. The ~/.zshrc line ensures the certificate path is remembered every time you open a new Terminal window.

Start Claude Code

In Terminal, type this and press Return:

claude

The first time you run it, it will ask for your API key. Paste the key from Step 4 and press Return.

✓

Using Claude Code Going Forward

  1. Open Terminal (Command + Space, type Terminal)
  2. Type cd (with a space), then drag your project folder into the Terminal window and press Return
  3. Type claude and press Return
Tip: You only need to install everything once. Next time, just open Terminal and type claude.
